# JobSonar

Система автоматического мониторинга вакансий на HH.ru с доставкой в Telegram.

Каждые 20–45 минут боты обходят выдачу hh.ru, фильтруют вакансии по своим профилям (роль, уровень, зарплата, формат работы, география, отрасль) и присылают подходящие в Telegram-каналы. Каждая вакансия отправляется только один раз — дедупликация через SQLite.

> ⚠️ **Актуальный код живёт в папке [`web/`](web/).** Файлы в корне репозитория — старая версия на `api.hh.ru`, который HH закрыл для соискателей (декабрь 2025 — апрель 2026). Корень оставлен для истории, запускать его бессмысленно — см. [README_DEPRECATED.md](README_DEPRECATED.md).

---

## Как это работает

HH.ru закрыл публичный API, поэтому JobSonar парсит **обычные страницы поиска** `hh.ru/search/vacancy`:

1. Запрос со страницей выдачи делается через `requests` с браузерными заголовками (`BROWSER_HEADERS`).
2. Внутри HTML hh.ru встраивает SSR-стейт (большой JSON, Redux store) — он извлекается по маркеру `data-ssr-state-length` через `json.JSONDecoder().raw_decode()`.
3. Из стейта берётся `vacancySearchResult.vacancies[]`, каждая вакансия нормализуется (`_normalize_vacancy`) в формат старого API: `name`, `employer`, `salary`, `area`, `schedule`, `work_format`, `experience`, `published_at`.
4. Дальше — фильтры конкретного бота и отправка в Telegram.

Никакого Selenium/Playwright — только `requests` (сервер с 1 ГБ RAM браузер не потянет).

### Защита от анти-бота hh.ru

hh.ru флагует сессию (не IP) за слишком частые запросы и начинает отдавать 403 либо заглушку `/vpncheeck` (она приходит с кодом 200 — ловится по URL редиректа). Встроенная защита в `web/utils.py`:

- **Троттлинг** — глобальная пауза 1–2 секунды (с джиттером) между *любыми* запросами к hh.ru;
- **Сброс сессии** — при первом 403/vpncheeck: очистка cookies → пауза 20–40 секунд → «прогрев» заходом на `hh.ru/` → повтор запроса. Не чаще одного раза в 5 минут, чтобы не долбиться, если не помогло;
- Свежая сессия с того же IP сразу получает 200 — проверено живыми тестами.

### Известные особенности парсинга

- `snippet` (описание) в списочной выдаче **всегда пустой** — фильтровать можно только по названию вакансии и атрибутам;
- URL-параметр `work_format=remote` — это *подсказка*, а не строгий фильтр (hh.ru подмешивает офисные вакансии), поэтому каждый бот дополнительно проверяет формат работы в данных вакансии;
- при поиске русских подстрок учитывается е/ё: ищем корень `'удал'`, а не `'удален'` (hh.ru пишет «Удалённо»).

## Архитектура

5 независимых процессов, у каждого свой Telegram-токен, канал и SQLite-база:

| Бот | Файл | Канал (.env) | Кого ищет |
|---|---|---|---|
| **HR + COO** | `web/main.py` | `TG_TOKEN_HR` | Руководящие HR-позиции и операционные директора |
| **Analyst** | `web/main_analyst.py` | `TG_TOKEN_TECH` | Бизнес/дата/продуктовые/системные аналитики |
| **Recruiter** | `web/main_recruiter.py` | `TG_TOKEN_RECRUITER` | Линейные IT-рекрутеры, сорсеры, T&D |
| **Sales** | `web/main_sales.py` | `TG_TOKEN_SALES` | B2B-продажи в digital/IT, только удалёнка |
| **Monitor** | `web/monitor.py` | `MONITOR_TOKEN` | Сторож: статусы ботов по команде `/status` |

Общий код — в `web/utils.py` (парсинг, троттлинг, форматирование зарплаты/даты, фильтр «работодатель-физлицо», Telegram-отправка, умный сон) и `web/db.py` (SQLite-дедупликация).

`web/whitelist.py` — `APPROVED_COMPANIES`: ~800 одобренных работодателей с категориями (🏆 гиганты / 🥇 крупные / 🥈 средние / 🥉 небольшие). Вакансии от них помечаются эмодзи категории и попадают в «Топ компании» в итогах дня.

### Цикл каждого бота

1. **Поиск по whitelist** (HR, Analyst): ID компаний батчами по 20 штук → два запроса на батч (удалёнка + целевые города);
2. **Глобальный поиск**: по ключевым словам профиля (`search_field=name` — только в названии вакансии);
3. Фильтрация (см. логику ботов ниже) → отправка новых вакансий в Telegram → пометка в БД;
4. **Итоги дня**: после 23:00 МСК один раз в сутки — сводка «Топ компании / Остальные / Всего»;
5. **Умный сон** (`get_smart_sleep_time`): будни — цикл каждые 20–30 минут, ночь (23:00–7:00) — сон до утра, выходные — реже, вечер воскресенья — сон до понедельника. У Recruiter свой график (15–30 минут днём, ночью раз в час).

Во время сна бот каждые 10 секунд проверяет Telegram: сообщение со словом «стоп» в канале останавливает процесс.

## Логика фильтрации по ботам

### Общая логика зарплаты (одинакова для всех)

| Что указано в вакансии | Решение |
|---|---|
| Зарплата не указана | **Показать** (хорошие зарплаты часто скрывают), в сообщении `-` |
| RUR ниже порога бота | **Скипнуть** (явно низкая) |
| RUR выше порога | Показать, сумма **жирным** |
| USD / EUR | Всегда показать **жирным** |

### 1. HR + COO Bot (`web/main.py`, v8.0)

Один процесс, два профиля (`config.py` + `config_coo.py`), общий канал и база. COO-профиль обрабатывается первым (приоритет при дедупликации).

**Попадание в выборку** — одно из двух:
- *Прямое совпадение*: название содержит фразу из `direct_titles` («HR Director», «HRD», «HRBP», «Операционный директор», «COO», «Chief of Staff»…);
- *Комбо*: уровень из `role_levels` («директор», «head», «руководитель»…) **И** контекст из `hr_context`/`role_context` («персонал», «talent», «c&b» — для HR; «операци», «executive», «бизнес-процесс» — для COO).

**Отсев:**
- стоп-слова в названии: у HR — продажи, клиенты, кредиты, логистика, стройка, юристы/финансы, маркетинг, админ, IT-технические роли («персональный менеджер» не пройдёт); у COO — другие C-level (CFO, CTO, CMO…), сферы без удалёнки (HoReCa, ритейл, клиники, фитнес), логистика, производство, джуны/ассистенты;
- вакансии без опыта (`noExperience`);
- **география**: Москва / Н.Новгород (`TARGET_AREAS = 1, 66`) — любой формат работы; все остальные регионы — только явная удалёнка;
- **зарплатные пороги**: HR — 200к (250к в глобальном поиске), COO — 250к везде.

**Маркер 🔥** — компания из whitelist + чистая удалёнка (без офисных/гибридных пометок).

### 2. Analyst Bot (`web/main_analyst.py`, v7.1)

**Попадание:** название содержит ключевик профиля («Бизнес-аналитик», «Data Analyst», «Продуктовый аналитик», «BI», «Системный аналитик»…).

**Отсев:**
- стоп-слова: разработчики, тестировщики, инженеры, архитекторы, админы, джуны/стажёры, 1С, SAP, консультанты;
- дополнительный жёсткий блок dev-ролей в названии («разработки», «developer», «backend», «golang»…);
- **география**: *системные аналитики* — только чистая удалёнка; остальные — удалёнка откуда угодно, офис/гибрид только вне чёрного списка регионов (`BLACKLISTED_AREAS` — Казахстан и др.);
- порог зарплаты: 200к RUR.

**Навыки**: из названия вакансии извлекаются `target_skills` (SQL, Python, BPMN, ETL, Jira…) — только для отображения строкой 🛠 в сообщении, **не для фильтрации** (требование навыков в названии отсекало ~90% реальных вакансий — выпилено).

### 3. Recruiter Bot (`web/main_recruiter.py`, v1.4) — «Ultra-Strict»

Только глобальный поиск по ключевикам («IT Recruiter», «Сорсер», «Talent Acquisition», «Внутренние коммуникации», «T&D», «DevRel»…), запрос с `work_format=remote`.

**Отсев:**
- **руководство отрезается**: «director», «head», «lead», «senior», «ведущий» — фокус строго на линейных исполнителях;
- не те роли: HRG, HRBP, бренд, C&B, КДП/кадровое делопроизводство, пиар/маркетинг/финансы/IT-аналитика;
- **работодатели-физлица** (`is_individual_person`): ИП, ФИО с инициалами, отчества в названии — типичные «частные рекрутёры» отсекаются;
- **только чистая удалёнка**: любой намёк на офис/гибрид — скип;
- стоп-домены: казино, крипта, форекс, вебкам, МФО/коллекторы + гео-блок локальных рынков СНГ (Казахстан, Минск, Тбилиси, Ереван, Баку…);
- порог зарплаты: 100к RUR;
- анти-спам: одна и та же пара «работодатель + название» в рамках цикла отправляется один раз.

### 4. Sales Bot (`web/main_sales.py`, v1.3)

Только глобальный поиск по ключевикам («Менеджер по продажам», «Account Manager», «KAM», «BizDev», «Head of Sales», «CSM»…), `work_format=remote`, период 7 дней.

**Отсев:**
- стоп-слова в названии: линейный персонал (операторы, кассиры, торговые представители), product/project, вакансии с требованием иностранных языков;
- **работодатели-физлица** — как у Recruiter;
- **только чистая удалёнка** — офис/гибрид/«на месте» отсекаются жёстко;
- **стоп-сферы** (`stop_domains`, главный фильтр): логистика/карго/ВЭД, недвижимость (включая Бали/Дубай), крипта/форекс/казино/MLM, юридический масс-маркет (банкротство, взыскание), стройка/ремонт/клининг, промышленность и B2B-товарка, FMCG/ритейл/HoReCa, фарма/медицина/туризм, франшизы. Цель — оставить продажи digital/IT-продуктов и услуг;
- порог зарплаты: 100к RUR.

Маркер: 🔥 — компания из whitelist, 🤝 — остальные.

### 5. Monitor Bot (`web/monitor.py`)

Не ищет вакансии. По команде `/status` (или кнопке «🔄 Обновить статус») показывает состояние всех 4 ботов: жив ли процесс + последний статус из `status_*.txt` (что бот сейчас делает / когда проснётся / сколько отправлено за день). Если процесс упал — шлёт авто-тревогу в админский чат с хвостом лога. Раз в 30 минут подрезает логи больше 5 МБ.

## Установка

### Требования

- Python 3.10+
- ~100 МБ RAM на бота (суммарно 5 ботов спокойно живут на VPS с 1 ГБ)
- Сервер с IP, с которого доступен hh.ru и api.telegram.org

```bash
git clone https://github.com/dmsgord/JobSonar.git
cd JobSonar/web
pip install requests python-dotenv pyTelegramBotAPI
```

### .env

Создать `web/.env` (в git не хранится):

```env
# HR + COO бот (общий канал)
TG_TOKEN_HR=123456:ABC...
TG_CHAT_ID_HR=-100...

# Analyst
TG_TOKEN_TECH=...
TG_CHAT_ID_TECH=...

# Recruiter
TG_TOKEN_RECRUITER=...
TG_CHAT_ID_RECRUITER=...

# Sales
TG_TOKEN_SALES=...
TG_CHAT_ID_SALES=...

# Monitor
MONITOR_TOKEN=...
ADMIN_CHAT_ID=...
```

Токены создаются через [@BotFather](https://t.me/BotFather), `chat_id` — ID канала/группы, куда бот добавлен.

### Локальный запуск (проверка)

```bash
cd web
python main.py            # HR+COO
python main_analyst.py    # Analyst
python main_recruiter.py  # Recruiter
python main_sales.py      # Sales
python monitor.py         # Monitor
```

SQLite-базы (`jobsonar_*.db`), логи (`log_*.txt`) и статус-файлы (`status_*.txt`) создаются автоматически рядом с кодом.

> Если бот пишет в лог `vpncheeck` — hh.ru не нравится ваш IP (VPN/зарубежный хостинг). Нужен IP, с которого hh.ru открывается в браузере без проверок.

### Деплой на сервер (systemd)

На каждый бот — свой юнит. Пример `/etc/systemd/system/jobsonar-hr.service`:

```ini
[Unit]
Description=JobSonar HR Bot
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=root
WorkingDirectory=/root/JobSonar/web
ExecStart=/usr/bin/python3 -u main.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

Аналогично `jobsonar-analyst` (`main_analyst.py`), `jobsonar-recruiter` (`main_recruiter.py`), `jobsonar-sales` (`main_sales.py`), `jobsonar-monitor` (`monitor.py`).

> ⚠️ `WorkingDirectory` обязан указывать на **`web/`**, а не на корень репозитория — в корне лежит мёртвая API-версия с теми же именами файлов.

```bash
systemctl daemon-reload
systemctl enable --now jobsonar-hr jobsonar-analyst jobsonar-recruiter jobsonar-sales jobsonar-monitor
```

Обновление:

```bash
cd ~/JobSonar && git pull
systemctl restart jobsonar-hr jobsonar-analyst jobsonar-recruiter jobsonar-sales jobsonar-monitor
```

Проверка: `/status` Monitor-боту в Telegram, либо `journalctl -u jobsonar-hr -f` на сервере.

## Настройка под себя

Вся логика «кого ищем» — в `config_*.py`, код трогать не нужно:

- `keywords` — поисковые запросы к hh.ru (ищутся в названии вакансии);
- `direct_titles` / `role_levels` + контекст — правила попадания (HR/COO);
- `stop_words` / `stop_words_title` — отсев по названию;
- `stop_domains` — отсев по отрасли (Sales/Recruiter);
- `MIN_SALARY`, `TARGET_AREAS`, `SEARCH_PERIOD` — пороги, города, глубина поиска в днях;
- `whitelist.py` — свой список приоритетных компаний (ID работодателя берётся из URL `hh.ru/employer/<id>`).
